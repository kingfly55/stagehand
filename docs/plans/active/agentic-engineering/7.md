# Milestone 7 — Agent Review Loop

## Scope
Build an automated PR review system: a Claude Code skill that reviews PRs for divergence impact, golden principle violations, test status, and description quality. Implement auto-labeling of PRs by divergence impact category. Integrate with the adversarial review pipeline from AGENT_ORCHESTRATION.md.

## Files to Create/Modify
- `.claude/skills/pr-review.md` — new; Claude Code skill for automated PR review
- `scripts/pr-review.sh` — new; script that performs automated PR checks
- `scripts/pr-label.sh` — new; script that determines and applies PR labels
- `.github/workflows/pr-review.yml` — new; GitHub Actions workflow for automated review on PR
- `claude.md` — add pointer to PR review skill

## Implementation Steps
1. **Create `scripts/pr-review.sh`:**
   - Accept a PR branch name or ref as argument.
   - **Check 1 — Divergence impact**: `git diff --name-only main..<branch>` and cross-reference with `docs/fork/DIVERGENCE.md`. Report any upstream files being modified that aren't in DIVERGENCE.md.
   - **Check 2 — Golden principle violations**: Check if any staged files match protected paths from `docs/dev/GOLDEN_PRINCIPLES.md` (parse the protected paths list).
   - **Check 3 — Typecheck**: Run `pnpm typecheck` and report pass/fail.
   - **Check 4 — Tests**: Run `pnpm build:esm && pnpm test:core` and report pass/fail.
   - **Check 5 — PR description quality**: Check if the PR body (from `gh pr view`) contains: a problem statement, what changed, and how it was verified. Flag if any section is missing.
   - Output a structured review summary in markdown format.
   - Make executable.

2. **Create `scripts/pr-label.sh`:**
   - Accept a PR number as argument.
   - Determine the label based on changed files:
     - `upstream-safe`: No files in `docs/fork/DIVERGENCE.md` are touched, no protected paths touched.
     - `divergence-update`: Modifies files listed in DIVERGENCE.md or adds new fork-specific files.
     - `needs-human-review`: Touches protected paths (understudy/*, handlers/*), modifies golden principles, or modifies CI/workflow files.
   - Apply the label via `gh pr edit --add-label`.
   - Make executable.

3. **Create `.github/workflows/pr-review.yml`:**
   - Trigger on `pull_request` events (opened, synchronize).
   - Steps:
     1. Checkout code
     2. Install dependencies (`pnpm install`)
     3. Run `scripts/pr-review.sh` with the PR branch
     4. Run `scripts/pr-label.sh` with the PR number
     5. Post the review summary as a PR comment via `gh pr comment`
   - Use appropriate permissions for PR comments and labels.

4. **Create `.claude/skills/pr-review.md`:**
   - Skill name: `pr-review`
   - Description: Review a PR for divergence impact, golden principle compliance, and quality.
   - Instructions:
     1. Identify the PR branch (from context or argument).
     2. Run `bash scripts/pr-review.sh <branch>`.
     3. Review the output and add any additional observations about code quality, architecture alignment, or potential issues.
     4. If golden principle violations are found, flag them prominently and suggest remediation.
     5. Determine the appropriate label and explain why.
     6. Post findings as a PR comment.
   - References: `docs/dev/GOLDEN_PRINCIPLES.md`, `docs/fork/DIVERGENCE.md`, `docs/fork/MERGE_POLICY.md`.

5. **Update `claude.md`** to reference the PR review skill and workflow.

6. **Create GitHub labels** (document the required labels in the workflow file comments):
   - `upstream-safe`
   - `divergence-update`
   - `needs-human-review`

## Verification Steps
1. Syntax-check all scripts:
   ```bash
   bash -n scripts/pr-review.sh && echo "OK: pr-review.sh" || echo "FAIL"
   bash -n scripts/pr-label.sh && echo "OK: pr-label.sh" || echo "FAIL"
   ```
2. Validate the workflow YAML:
   ```bash
   python3 -c "import yaml; yaml.safe_load(open('.github/workflows/pr-review.yml'))" && echo "OK" || echo "FAIL"
   ```
3. Confirm the review script checks all required areas:
   ```bash
   grep -c "DIVERGENCE\|GOLDEN_PRINCIPLES\|typecheck\|test:core\|description" scripts/pr-review.sh
   # Should be >= 5
   ```
4. Confirm the label script supports all three labels:
   ```bash
   grep -c "upstream-safe\|divergence-update\|needs-human-review" scripts/pr-label.sh
   # Should be >= 3
   ```
5. Confirm skill file exists and references key docs:
   ```bash
   grep -c "GOLDEN_PRINCIPLES\|DIVERGENCE\|MERGE_POLICY" .claude/skills/pr-review.md
   # Should be >= 3
   ```
6. Run `pnpm typecheck` and confirm zero errors
