# Milestone 1 — Upstream Sync Infrastructure

## Scope
Establish the foundational upstream tracking infrastructure: add the root-level `pnpm typecheck` command (required by all milestones), configure the `upstream` git remote, create a divergence tracking document, build a merge-from-upstream helper script, and add a CI check that warns when the fork falls behind upstream.

## Files to Create/Modify
- `.git/config` (via `git remote add`) — add `upstream` remote
- `docs/fork/DIVERGENCE.md` — new; living catalog of every intentional fork divergence
- `scripts/upstream-sync.sh` — new; script to fetch upstream, identify conflicts, provide merge context
- `.github/workflows/upstream-check.yml` — new; CI workflow that warns when upstream is >N commits ahead
- `claude.md` — minor update to confirm `upstream` remote instructions are accurate
- `package.json` — add `"typecheck": "turbo run typecheck"` to root scripts (see Prerequisite Notes in plan.md)
- `turbo.json` — add `"typecheck"` task definition

## Implementation Steps
1. **Add root-level `typecheck` command** (critical prerequisite for all milestones):
   - Add `"typecheck": "turbo run typecheck"` to the `"scripts"` section of root `package.json`.
   - Add a `"typecheck"` task inside the `"tasks"` object of `turbo.json` (similar to the existing `"format"` task, NOT `"lint"` which has unwanted `dependsOn` and `inputs`):
     ```json
     "typecheck": {
       "outputs": [],
       "cache": false
     }
     ```
   - Verify: `pnpm typecheck` should now run typecheck in all workspace packages and exit 0. **Note:** Each workspace package must already have a `"typecheck"` script in its own `package.json` for turbo to find it — verify with `pnpm -r run typecheck` first.
   - **This must be done before any other step** so that all subsequent verification commands work.

2. **Add the upstream remote (idempotent):**
   ```bash
   # Check if upstream already exists before adding
   if ! git remote get-url upstream &>/dev/null; then
     git remote add upstream https://github.com/browserbase/stagehand.git
   fi
   git fetch upstream
   ```
   **Note:** `git remote add` will fail with a fatal error if the remote already exists. Always guard with a check.
3. **Create directories** if they don't exist:
   ```bash
   mkdir -p docs/fork
   mkdir -p scripts
   ```
4. **Create `docs/fork/DIVERGENCE.md`:**
   - Run `git log --oneline main..native-base` (or equivalent) to identify fork-only commits.
   - Run `git diff --name-status upstream/main..HEAD` to identify all files that differ (use HEAD, not origin/main, since the working branch is `native-base` not `main`).
   - For each diverged file, document: file path, type of change (added/modified/deleted), rationale, and which side to prefer during merge conflicts.
   - Include a "Last verified" date field at the top.
   - **Filter out** fork-only paths (`docs/`, `scripts/`, `.claude/`, `.husky/`, `.pipeline-venv/`, `.changeset/`) — DIVERGENCE.md should focus on files that overlap with upstream. Also filter out root-level `.md` files that are fork-only planning docs (they will be moved in Milestone 5).
5. **Create `scripts/upstream-sync.sh`:**
   - Must start with `#!/usr/bin/env bash` and `set -euo pipefail`.
   - **Git version check:** The script uses `git merge-tree --write-tree` which requires git ≥ 2.38. Add an early check. **Do NOT use `bc`** — it may not be installed on minimal Ubuntu. Use `sort -V` instead:
     ```bash
     git_version=$(git version | grep -oP '\d+\.\d+(\.\d+)?' | head -1)
     min_version="2.38"
     if [ "$(printf '%s\n' "$min_version" "$git_version" | sort -V | head -1)" != "$min_version" ]; then
       echo "ERROR: git >= 2.38 required for merge-tree --write-tree (found $git_version)"
       exit 1
     fi
     ```
   - Checks that working tree is clean (`git diff --quiet && git diff --cached --quiet`) before proceeding; aborts with a message if dirty.
   - Fetches `upstream main`.
   - Runs `git log --oneline upstream/main..HEAD` to show fork-only commits.
   - Runs `git log --oneline HEAD..upstream/main` to show how far behind.
   - To preview conflicts **without modifying the working tree**, use `git merge-tree`:
     ```bash
     # Safe conflict preview (git 2.38+):
     # Exit code 0 = clean merge, 1 = conflicts. With set -e, exit code 1
     # would kill the script, so capture it explicitly:
     merge_output=$(git merge-tree --write-tree HEAD upstream/main 2>&1) || true
     merge_rc=$?
     # Parse CONFLICT lines from $merge_output
     ```
     **Note:** `git merge-tree --write-tree` exits 1 if conflicts are found. Since the script uses `set -e`, you MUST capture the exit code with `|| true` or the script will terminate before it can report the conflicts.
     **Do NOT use `git merge upstream/main --no-commit`** for preview — it modifies the working tree and index, and a failed `git merge --abort` can corrupt state.
   - On conflict, cross-references `docs/fork/DIVERGENCE.md` and prints guidance for each conflicting file (prefer fork side if listed in DIVERGENCE.md, prefer upstream otherwise).
   - Make the script executable (`chmod +x`).
   - **Edge case:** Handle the "already up to date" case (0 commits behind) gracefully with an informational message and exit 0.
6. **Create `.github/workflows/upstream-check.yml`:**
   - Scheduled workflow (weekly cron) **and** on `pull_request` to `native-base`.
   - Must `fetch-depth: 0` in the checkout step so full history is available for `git log` comparisons.
   - Fetches upstream, counts commits behind, and posts a warning annotation if >20 commits behind.
   - Must include `permissions: contents: read` and not require write access (read-only for scheduled checks).
   - **Cron note:** GitHub Actions disables scheduled workflows after 60 days of repo inactivity. Document this limitation.
   - **YAML key note:** Use `"on":` (quoted) instead of bare `on:` for the trigger key — `on` is a YAML boolean keyword and some parsers interpret bare `on` as `true`. Most GitHub Actions templates use the quoted form.
7. **Verify `claude.md`** references to `git pull upstream main` are now valid.

## Success Criteria
- [ ] `pnpm typecheck` works from the repo root (root script and turbo task exist)
- [ ] `git remote -v` shows `upstream` pointing to `https://github.com/browserbase/stagehand.git`
- [ ] `docs/fork/DIVERGENCE.md` exists and lists at least 5 diverged files with rationale and merge-preference columns
- [ ] `scripts/upstream-sync.sh` exists, is executable, starts with `set -euo pipefail`, and runs without error
- [ ] `scripts/upstream-sync.sh` does NOT use `git merge` to preview conflicts (must use `git merge-tree` or equivalent non-destructive method)
- [ ] `scripts/upstream-sync.sh` includes a git version check (≥ 2.38)
- [ ] `.github/workflows/upstream-check.yml` exists and is valid YAML
- [ ] `pnpm typecheck` passes with zero errors
- [ ] `pnpm build:esm` completes successfully

## Verification Steps
1. Confirm root-level typecheck works:
   ```bash
   grep -q '"typecheck"' package.json && echo "OK: root script exists" || echo "FAIL: no root typecheck script"
   grep -q '"typecheck"' turbo.json && echo "OK: turbo task exists" || echo "FAIL: no turbo typecheck task"
   pnpm typecheck && echo "OK: typecheck passes" || echo "FAIL: typecheck failed"
   ```
2. Run `git remote -v` and confirm output includes a line matching `upstream.*browserbase/stagehand`
3. Run `cat docs/fork/DIVERGENCE.md` and confirm it contains at least 5 file entries with rationale
4. Run `bash scripts/upstream-sync.sh` and confirm it exits 0 (or exits with an informational message, not an error)
5. Validate the workflow YAML (PyYAML may not be installed — use fallback):
   ```bash
   python3 -c "import yaml; yaml.safe_load(open('.github/workflows/upstream-check.yml'))" 2>/dev/null \
     && echo "OK: YAML valid" \
     || echo "WARN: PyYAML not available, falling back to structural check"
   ```
   At minimum, confirm the file is non-empty and contains `on:` and `jobs:`:
   ```bash
   grep -q "^on:" .github/workflows/upstream-check.yml && grep -q "^jobs:" .github/workflows/upstream-check.yml && echo "OK" || echo "FAIL"
   ```
   **Note:** The `on:` check uses `^on:` (start of line). If the YAML uses `"on":` (quoted key, common for GitHub Actions), also check: `grep -qE '^"?on"?:' .github/workflows/upstream-check.yml`
6. Run `pnpm typecheck` and confirm zero errors
7. Run `pnpm build:esm` and confirm zero errors
