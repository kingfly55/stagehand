# Milestone 1 — Upstream Sync Infrastructure

## Scope
Establish the foundational upstream tracking infrastructure: configure the `upstream` git remote, create a divergence tracking document, build a merge-from-upstream helper script, and add a CI check that warns when the fork falls behind upstream.

## Files to Create/Modify
- `.git/config` (via `git remote add`) — add `upstream` remote
- `docs/fork/DIVERGENCE.md` — new; living catalog of every intentional fork divergence
- `scripts/upstream-sync.sh` — new; script to fetch upstream, identify conflicts, provide merge context
- `.github/workflows/upstream-check.yml` — new; CI workflow that warns when upstream is >N commits ahead
- `claude.md` — minor update to confirm `upstream` remote instructions are accurate

## Implementation Steps
1. **Add the upstream remote:**
   ```bash
   git remote add upstream https://github.com/browserbase/stagehand.git
   git fetch upstream
   ```
2. **Create `docs/fork/` directory** if it doesn't exist.
3. **Create `docs/fork/DIVERGENCE.md`:**
   - Run `git log --oneline main..native-base` (or equivalent) to identify fork-only commits.
   - Run `git diff --name-status upstream/main..origin/main` to identify all files that differ.
   - For each diverged file, document: file path, type of change (added/modified/deleted), rationale, and which side to prefer during merge conflicts.
   - Include a "Last verified" date field at the top.
4. **Create `scripts/upstream-sync.sh`:**
   - Fetches `upstream main`.
   - Runs `git log --oneline upstream/main..HEAD` to show fork-only commits.
   - Runs `git log --oneline HEAD..upstream/main` to show how far behind.
   - Attempts a `git merge upstream/main --no-commit` in dry-run fashion to preview conflicts.
   - On conflict, cross-references `docs/fork/DIVERGENCE.md` and prints guidance for each conflicting file (prefer fork side if listed in DIVERGENCE.md, prefer upstream otherwise).
   - Make the script executable (`chmod +x`).
5. **Create `.github/workflows/upstream-check.yml`:**
   - Scheduled workflow (e.g., weekly cron or on PR).
   - Fetches upstream, counts commits behind, and posts a warning annotation if >20 commits behind.
6. **Verify `claude.md`** references to `git pull upstream main` are now valid.

## Success Criteria
- [ ] `git remote -v` shows `upstream` pointing to `https://github.com/browserbase/stagehand.git`
- [ ] `docs/fork/DIVERGENCE.md` exists and lists every file modified or added by the fork with rationale
- [ ] `scripts/upstream-sync.sh` exists, is executable, and runs without error
- [ ] `.github/workflows/upstream-check.yml` exists and is valid YAML
- [ ] `pnpm typecheck` passes with zero errors
- [ ] `pnpm build:esm` completes successfully

## Verification Steps
1. Run `git remote -v` and confirm output includes a line matching `upstream.*browserbase/stagehand`
2. Run `cat docs/fork/DIVERGENCE.md` and confirm it contains at least one file entry with rationale
3. Run `bash scripts/upstream-sync.sh` and confirm it exits 0 (or exits with an informational message, not an error)
4. Run `python3 -c "import yaml; yaml.safe_load(open('.github/workflows/upstream-check.yml'))"` (or equivalent) to validate the workflow YAML
5. Run `pnpm typecheck` and confirm zero errors
6. Run `pnpm build:esm` and confirm zero errors

## Dependencies
None — this is the first milestone.
