# Milestone 2 — Agent Map (claude.md Restructure)

## Scope
Restructure `claude.md` from a monolithic file into a short (~100-line) agent map with pointers to detailed docs. Create the `docs/` directory tree for agent-consumable knowledge. Extract API usage docs and verification commands into their own files. Ensure no information is lost.

## Files to Create/Modify
- `claude.md` — rewrite to be a concise map (<120 lines) with pointers
- `docs/dev/VERIFICATION.md` — new; extracted verification commands and test guidance
- `docs/api-usage.md` — new; extracted API usage documentation (Stagehand init, act, extract, observe, agent patterns)
- `docs/fork/ARCHITECTURE_DELTA.md` — new; placeholder for fork architecture differences
- `docs/fork/MERGE_POLICY.md` — new; placeholder for merge policy (populated in Milestone 4)
- `docs/dev/` — create directory
- `docs/plans/active/` — ensure directory exists
- `docs/plans/completed/` — ensure directory exists

## Implementation Steps
1. **Read the full current `claude.md`** and inventory every piece of information it contains.
2. **Create `docs/dev/VERIFICATION.md`:**
   - Move all verification commands (typecheck, build:esm, test:core, test:native, example commands).
   - Include the "inner loop" and "full validation" command sets.
   - Add notes on when to use each command.
3. **Create `docs/api-usage.md`:**
   - Move all Stagehand API documentation (initialization, act, extract, observe, agent, context, page patterns).
   - Include code examples.
   - Note that this is a summary — point to `packages/docs/` for full upstream docs.
4. **Create `docs/fork/ARCHITECTURE_DELTA.md`:**
   - Document the high-level architecture difference: Playwright-native mode vs CDP-only upstream.
   - Reference `DIVERGENCE.md` for file-level details.
5. **Create `docs/fork/MERGE_POLICY.md`:**
   - Stub file with a header and "See Milestone 4 for full merge policy."
6. **Ensure directory structure exists:**
   ```
   docs/
   ├── fork/
   │   ├── DIVERGENCE.md        (from Milestone 1)
   │   ├── MERGE_POLICY.md      (stub)
   │   └── ARCHITECTURE_DELTA.md
   ├── dev/
   │   └── VERIFICATION.md
   ├── api-usage.md
   └── plans/
       ├── active/
       └── completed/
   ```
7. **Rewrite `claude.md`:**
   - Keep: fork context (branch, repo, active feature), hard constraints, ground truth pointers.
   - Replace inline content with pointers: `→ docs/dev/VERIFICATION.md`, `→ docs/api-usage.md`, etc.
   - Add a "Documentation Map" section listing all docs/ files.
   - Target: under 120 lines.
8. **Verify no information loss** by diffing the old and new content coverage.

## Success Criteria
- [ ] `claude.md` is under 120 lines (verified by `wc -l claude.md`)
- [ ] `docs/dev/VERIFICATION.md` exists and contains all verification commands from old `claude.md`
- [ ] `docs/api-usage.md` exists and contains API usage patterns from old `claude.md`
- [ ] `docs/fork/ARCHITECTURE_DELTA.md` exists
- [ ] `docs/fork/MERGE_POLICY.md` exists (stub is acceptable)
- [ ] Every file path referenced in `claude.md` resolves to an actual file
- [ ] `pnpm typecheck` passes with zero errors

## Verification Steps
1. Run `wc -l claude.md` and confirm the count is ≤ 120
2. Run a script or manually verify that every path mentioned in `claude.md` (lines containing `docs/`) corresponds to an existing file:
   ```bash
   grep -oP 'docs/[^\s`)]+' claude.md | while read f; do [ -f "$f" ] && echo "OK: $f" || echo "MISSING: $f"; done
   ```
3. Confirm `docs/dev/VERIFICATION.md` contains the strings `pnpm typecheck`, `pnpm build:esm`, and `pnpm test:core`
4. Confirm `docs/api-usage.md` contains the strings `Stagehand`, `act`, `extract`, `observe`
5. Run `pnpm typecheck` and confirm zero errors
