## Verification Steps
1. Run `wc -l claude.md` and confirm the count is ≤ 120
2. Run a script or manually verify that every path mentioned in `claude.md` (lines containing `docs/`) corresponds to an existing file:
   ```bash
   # -oP uses Perl regex (available on Ubuntu with GNU grep)
   # Strip trailing punctuation (periods, commas, colons) that grep may capture
   # NOTE: Do NOT pipe into `while read` — the pipe creates a subshell and
   # failures won't propagate. Use process substitution or a temp variable.
   missing=0
   while read -r f; do
     if [ -f "$f" ]; then
       echo "OK: $f"
     else
       echo "MISSING: $f"
       missing=1
     fi
   done < <(grep -oP 'docs/[^\s`)\]>]+' claude.md | sed 's/[.,;:)]*$//' | sort -u)
   [ "$missing" -eq 0 ] && echo "ALL PATHS OK" || echo "FAIL: some paths missing"
   ```
   **Fail condition:** If ANY line prints "MISSING", the verification fails. The `missing` variable must be checked OUTSIDE the loop — using a pipe instead of process substitution `< <(...)` would create a subshell where `missing=1` is lost.
3. Confirm `docs/dev/VERIFICATION.md` contains the strings `pnpm typecheck`, `pnpm build:esm`, and `pnpm test:core`
4. Confirm `docs/api-usage.md` contains the strings `Stagehand`, `act`, `extract`, `observe`
5. Run `pnpm typecheck` and confirm zero errors
