# Milestone 4 — Upstream Merge Agent Skill

## Scope
Build a Claude Code skill (implemented as a documented script + skill definition) that automates upstream merges: fetches upstream, attempts merge, resolves conflicts using `DIVERGENCE.md` as context, runs verification, and prepares a PR. Fully populate `docs/fork/MERGE_POLICY.md` with the merge strategy.

## Files to Create/Modify
- `scripts/upstream-merge.sh` — new; the full merge automation script
- `docs/fork/MERGE_POLICY.md` — fully populate (was a stub from Milestone 2)
- `.claude/skills/upstream-merge.md` — new; Claude Code skill definition for the merge workflow
- `docs/fork/DIVERGENCE.md` — may be updated if merge introduces new fork changes

## Implementation Steps
1. **Create `scripts/upstream-merge.sh`:**
   - Must start with `#!/usr/bin/env bash` and `set -euo pipefail`.
   - Accept optional arguments: `--dry-run` (preview only), `--auto-resolve` (apply DIVERGENCE.md rules automatically).
   - **Pre-flight checks (before any git operations):**
     - Verify working tree is clean: `git diff --quiet && git diff --cached --quiet` or abort with error.
     - Verify `upstream` remote exists: `git remote get-url upstream` or abort.
     - If `--dry-run`, perform ALL analysis using `git merge-tree` (no working-tree modifications) and exit.
   - Steps in the script:
     1. `git fetch upstream`
     2. Show commits behind: `git log --oneline HEAD..upstream/main | wc -l`
     3. If 0 commits behind, print "Already up to date" and exit 0.
     4. Show commits ahead (fork-only): `git log --oneline upstream/main..HEAD`
     5. If NOT `--dry-run`, attempt merge: `git merge upstream/main --no-commit --no-ff`
     6. If merge succeeds with no conflicts, proceed to verification.
     7. If conflicts:
        - Parse conflicting files from `git diff --name-only --diff-filter=U`
        - For each conflicting file, check if it appears in `docs/fork/DIVERGENCE.md`
        - If in DIVERGENCE.md with "prefer fork" → `git checkout --ours <file> && git add <file>`
        - If NOT in DIVERGENCE.md → `git checkout --theirs <file> && git add <file>`
        - **WARNING in output**: `checkout --ours`/`--theirs` discards the other side entirely. For files with interleaved changes, log a prominent warning: "⚠ File X resolved by taking fork side wholesale — upstream changes in this file are LOST. Review manually if upstream made important changes here."
        - If `--auto-resolve` is NOT set, abort and print the conflict list with guidance instead of auto-resolving.
        - Log each resolution decision with rationale.
     8. Stage all resolved files: `git add -A`
     9. Run verification: `pnpm typecheck && pnpm build:esm && pnpm test:core`
     10. If verification **fails**, abort the merge: `git merge --abort` and exit 1 with error details.
     11. If verification passes, commit with `--no-verify` (merge commits touch protected upstream files, which the pre-commit hook would block):
        ```bash
        git commit --no-verify -m "chore: merge upstream/main (N commits)

        Upstream changes: [summary]
        Conflict resolutions: [list]
        Verification: typecheck ✓, build ✓, tests ✓
        --no-verify reason: merge commit touches protected upstream files"
        ```
        **IMPORTANT:** The `--no-verify` is required because merge commits modify protected paths (upstream files in `understudy/`, `handlers/`). The commit message MUST document the reason per golden principles.
   - **Error handling:** Trap EXIT to run `git merge --abort` if script exits unexpectedly mid-merge:
     ```bash
     merge_in_progress=false
     cleanup() {
       if [ "$merge_in_progress" = true ]; then
         echo "ERROR: Script interrupted during merge. Aborting merge..."
         git merge --abort 2>/dev/null || true
       fi
     }
     trap cleanup EXIT
     # Set before starting merge:
     merge_in_progress=true
     git merge upstream/main --no-commit --no-ff
     # ... after successful commit:
     merge_in_progress=false
     trap - EXIT
     ```
     **Note:** The trap guard (`merge_in_progress`) prevents `git merge --abort` from running during pre-flight or post-commit phases where there's no merge to abort.
   - Make executable.
   - **Git version note:** If using `--dry-run` with `git merge-tree --write-tree`, the same git ≥ 2.38 requirement from Milestone 1 applies. Include the same version check (use `sort -V`, not `bc`). Also, `git merge-tree --write-tree` exits 1 on conflicts — with `set -e` active, capture the exit code explicitly: `merge_output=$(git merge-tree --write-tree HEAD upstream/main 2>&1) || true; merge_rc=$?`.

   **Edge cases to handle:**
   - `pnpm-lock.yaml` conflicts: These cannot be resolved with `--ours`/`--theirs` (the result would be invalid YAML). The script must detect this file in the conflict list, then: (a) accept upstream's `pnpm-lock.yaml` first via `git checkout --theirs pnpm-lock.yaml`, (b) run `pnpm install --no-frozen-lockfile` to reconcile with the merged `package.json` (uses `--no-frozen-lockfile` because the lockfile won't match the merged package.json), (c) stage the regenerated file: `git add pnpm-lock.yaml`. **Note:** If `package.json` also conflicts, resolve that FIRST (manually or with a merge), then handle the lockfile.
   - `package.json` conflicts: Flag for manual review — auto-resolution risks losing dependency changes from either side.
   - **`--ours`/`--theirs` semantics:** In a `git merge upstream/main` context, `--ours` = the current branch (fork), `--theirs` = upstream. Document this explicitly in the script comments to avoid confusion — the semantics are reversed during `git rebase`.

2. **Create `.claude/skills/upstream-merge.md`** (create directory first: `mkdir -p .claude/skills/`):
   - Skill name: `upstream-merge`
   - Description: Merges upstream stagehand changes into the fork
   - Instructions: When invoked, run `scripts/upstream-merge.sh`, review the output, and if there are complex conflicts that the script couldn't auto-resolve, manually resolve them using DIVERGENCE.md as context.
   - Post-conditions: All verification commands must pass.

3. **Fully populate `docs/fork/MERGE_POLICY.md`:**
   - Merge frequency recommendation (weekly or on significant upstream releases)
   - Conflict resolution hierarchy:
     1. `pnpm-lock.yaml` → delete conflicted file, run `pnpm install`, stage result
     2. `package.json` → always flag for manual review (both sides may add dependencies)
     3. Files in DIVERGENCE.md with "prefer fork" → keep ours
     4. Files not in DIVERGENCE.md → prefer upstream
     5. Files in both with structural/interleaved conflicts → flag for human review
   - When to update DIVERGENCE.md (any time a merge requires keeping fork-specific changes in a file not previously tracked)
   - Verification requirements post-merge (typecheck, build, test)
   - Rollback procedure if verification fails (`git merge --abort` if uncommitted, `git reset --hard HEAD~1` if committed)
   - Document the `--no-verify` escape hatch for merge commits that touch protected paths

4. **Test with a dry run** against current upstream state.

## Verification Steps
1. **Dry-run safety check:** Capture git status before and after to confirm no working-tree modification:
   ```bash
   before=$(git status --porcelain)
   bash scripts/upstream-merge.sh --dry-run
   after=$(git status --porcelain)
   [ "$before" = "$after" ] && echo "OK: no working tree changes" || echo "FAIL: dry-run modified working tree"
   ```
2. Confirm `scripts/upstream-merge.sh` is executable: `test -x scripts/upstream-merge.sh && echo "OK"`
3. Confirm script has error handling: `grep -q 'set -e' scripts/upstream-merge.sh && echo "OK" || echo "FAIL"`
4. Confirm `.claude/skills/upstream-merge.md` exists: `test -f .claude/skills/upstream-merge.md && echo "OK"`
5. Run `wc -l docs/fork/MERGE_POLICY.md` and confirm >50 lines
6. Confirm `docs/fork/MERGE_POLICY.md` contains the strings "conflict", "DIVERGENCE", "rollback", "pnpm-lock", and "package.json":
   ```bash
   for term in conflict DIVERGENCE rollback pnpm-lock package.json; do
     grep -qi "$term" docs/fork/MERGE_POLICY.md && echo "OK: $term" || echo "FAIL: $term"
   done
   ```
7. Confirm merge script handles pnpm-lock.yaml conflicts: `grep -q "pnpm-lock" scripts/upstream-merge.sh && echo "OK" || echo "FAIL: no lockfile handling"`
8. Confirm merge script has EXIT trap: `grep -q "trap" scripts/upstream-merge.sh && echo "OK" || echo "FAIL: no cleanup trap"`
9. Confirm merge script documents `--ours`/`--theirs` semantics: `grep -qi "ours.*fork\|theirs.*upstream" scripts/upstream-merge.sh && echo "OK" || echo "WARN: ours/theirs semantics not documented in comments"`
10. Confirm merge script uses `--no-verify` for commits: `grep -q '\-\-no-verify' scripts/upstream-merge.sh && echo "OK" || echo "FAIL: merge commit must use --no-verify to bypass protected path hooks"`
11. Run `pnpm typecheck` and confirm zero errors
