## Implementation Steps
1. **Verify which planning documents exist at repo root:**
   ```bash
   ls -1 *.md | grep -v README | grep -v CHANGELOG | grep -v TESTING_GUIDE
   ```
   Identify all planning docs that should be moved.

2. **Move each planning document** using `git mv` to preserve history:
   ```bash
   git mv PLAYWRIGHT_NATIVE_PLAN.md docs/plans/completed/
   git mv PHASE4_FINAL_BRIEF.md docs/plans/completed/
   git mv PHASE4_ADDENDUM.md docs/plans/completed/
   git mv V2_vs_V3_NATIVE_COMPARISON.md docs/plans/completed/
   git mv SNAPSHOT_UPGRADE_RESEARCH.md docs/plans/completed/
   git mv AGENT_ORCHESTRATION.md docs/plans/completed/
   git mv DOCS_UPDATE_PLAN.md docs/plans/completed/
   ```
   Note: If any file doesn't exist, skip it. If additional planning docs are found at root, move those too.

3. **Create `docs/plans/completed/INDEX.md`:**
   - Title: "Completed Plans"
   - Table with columns: Plan, Original Purpose, Completion Date (approximate)
   - One row per moved document with a brief description.

4. **Create `docs/plans/active/INDEX.md`:**
   - Title: "Active Plans"
   - Table with columns: Plan, Status, Owner
   - Include the current `agentic-engineering` plan as an entry.

5. **Create `docs/plans/TEMPLATE.md`:**
   ```markdown
   # Plan: [Title]

   ## Problem Statement
   [What problem does this solve?]

   ## Scope
   [What is in scope and out of scope?]

   ## Approach
   [High-level approach and key decisions]

   ## Milestones
   [Break into verifiable milestones]

   ## Verification
   [How to verify the plan is complete]

   ## Acceptance Criteria
   - [ ] [Criterion 1]
   ```

6. **Update `claude.md`** to include a pointer to `docs/plans/` and note that planning docs are no longer at repo root.

7. **Update any cross-references** in existing docs that pointed to root-level plan files (e.g., `claude.md` references to `PLAYWRIGHT_NATIVE_PLAN.md`).

## Verification Steps
1. Check no planning docs at root:
   ```bash
   ls *.md 2>/dev/null | grep -v README | grep -v CHANGELOG | grep -v TESTING_GUIDE | grep -v claude
   # Should produce empty output or only non-planning files
   ```
2. Check completed plans directory:
   ```bash
   ls docs/plans/completed/*.md | wc -l
   # Should be >= 8 (7 moved files + INDEX.md)
   ```
3. Confirm index files exist:
   ```bash
   test -f docs/plans/completed/INDEX.md && echo "OK" || echo "FAIL"
   test -f docs/plans/active/INDEX.md && echo "OK" || echo "FAIL"
   ```
4. Confirm template exists and has required sections:
   ```bash
   grep -c "Problem Statement\|Scope\|Approach\|Verification\|Acceptance Criteria" docs/plans/TEMPLATE.md
   # Should be >= 5
   ```
5. Confirm `claude.md` references plans:
   ```bash
   grep -q "docs/plans" claude.md && echo "OK" || echo "FAIL"
   ```
6. Run `pnpm typecheck` and confirm zero errors
