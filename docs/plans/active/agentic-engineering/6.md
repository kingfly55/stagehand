# Milestone 6 — Quality Grading & Entropy Management

## Scope
Create a quality scoring system and automated "doc gardening" checks to detect drift, stale documentation, broken pointers, and divergence inaccuracies. Deliver a quality score document, a gardening script, and a cleanup sweep skill for agents.

## Files to Create/Modify
- `docs/QUALITY_SCORE.md` — new; quality baseline tracking tech debt, doc freshness, divergence health
- `scripts/doc-gardening.sh` — new; automated checks for stale docs, broken pointers, divergence accuracy
- `.claude/skills/cleanup-sweep.md` — new; Claude Code skill for entropy identification and remediation
- `claude.md` — add pointer to `docs/QUALITY_SCORE.md`

## Implementation Steps
1. **Create `docs/QUALITY_SCORE.md`:**
   - Header with "Last assessed" date.
   - Section 1 — **Tech Debt Register**: table with columns: Item, Severity (high/medium/low), Description, Filed Date, Status.
     - Populate with known items (e.g., stub MERGE_POLICY if still incomplete, any TODO comments in code).
   - Section 2 — **Documentation Freshness**: table with columns: Document Path, Last Verified, Status (fresh/stale/unknown).
     - List every doc in `docs/` with its last verified date.
   - Section 3 — **Fork Divergence Health**: commits behind upstream, last sync date, DIVERGENCE.md accuracy status.
   - Section 4 — **Overall Score**: a simple grade (A/B/C/D) with criteria definitions.

2. **Create `scripts/doc-gardening.sh`:**
   - Must start with `#!/usr/bin/env bash` and `set -uo pipefail` (**not** `set -euo pipefail`). The script uses `set -u` for undefined variable safety and `set -o pipefail` for pipe error detection, but **omits `set -e`** because multiple checks are expected to detect failures and report them without aborting the script. Each check function should track its own pass/fail status and contribute to a final exit code.
   - **Check 1 — Pointer resolution**: Extract all `docs/` paths from `claude.md` (strip trailing punctuation), deduplicate, and verify each file exists. Print OK/MISSING for each. Use process substitution (`< <(grep ...)`) not pipe-to-while to avoid subshell variable loss.
   - **Check 2 — DIVERGENCE.md accuracy**:
     - First, `git fetch upstream` to ensure we have current upstream state. If `upstream` remote doesn't exist, skip this check with a warning (don't fail the whole script). Use: `git remote get-url upstream &>/dev/null || { echo "WARN: no upstream remote; skipping divergence check"; skip_divergence=true; }`
     - Run `git diff --name-only upstream/main..HEAD` to get actual diverged files.
     - Parse files listed in `docs/fork/DIVERGENCE.md`.
     - Report files that diverge but aren't documented, and files documented but no longer diverging.
     - **Filter out** from the divergence check — these are fork-only by nature and don't need tracking in DIVERGENCE.md: `docs/`, `scripts/`, `.claude/`, `.husky/`, `.pipeline-venv/`, `.changeset/` paths, and any new root-level `.md` files not present in upstream.
   - **Check 3 — Stale docs**: For each `.md` file in `docs/`, check `git log -1 --format=%ci <file>` and flag if older than 30 days. **Note:** Git commit date reflects last edit, not last verification. The script should check for a "Last verified" string inside the file and use that date if present, falling back to git commit date. **Exclude** `docs/plans/active/agentic-engineering/` (plan files are actively being implemented and have their own tracking).
   - **Check 4 — Orphan detection**: Check for any `.md` files at repo root that look like planning docs (heuristic: contains "## Problem" or "## Deliverables" or "## Scope"). Exclude `README.md`, `CHANGELOG.md`, `TESTING_GUIDE.md`, `claude.md`. **Do NOT scan** `.pipeline-venv/`, `.changeset/`, `.github/`, `packages/`, or `node_modules/` directories (use `find . -maxdepth 1 -name '*.md'` to restrict to root only). The repo also has `.md` files in `.github/ISSUE_TEMPLATE/` and `packages/*/` — these are NOT orphans.
   - Print a summary with counts: N pointers OK, N missing, N stale docs, N divergence mismatches.
   - Exit 0 if all checks pass, exit 1 if any issues found.
   - Make executable.

3. **Create `.claude/skills/cleanup-sweep.md`** (directory should already exist from Milestone 4; if not, run `mkdir -p .claude/skills/`):
   - Skill name: `cleanup-sweep`
   - Description: Run the doc gardening script, review the output, and fix any issues found.
   - Instructions:
     1. Run `bash scripts/doc-gardening.sh` and capture output.
     2. For broken pointers: either create the missing file or fix the pointer in `claude.md`.
     3. For stale docs: review and update the "Last verified" date if content is still accurate, or update content.
     4. For DIVERGENCE.md mismatches: update DIVERGENCE.md to reflect current state.
     5. Update `docs/QUALITY_SCORE.md` with new assessment date and findings.
   - Post-conditions: `scripts/doc-gardening.sh` exits 0.

4. **Update `claude.md`** to reference `docs/QUALITY_SCORE.md` in the documentation map.

## Verification Steps
1. Confirm quality score doc has required sections:
   ```bash
   count=$(grep -c "Tech Debt\|Documentation Freshness\|Divergence Health\|Overall Score" docs/QUALITY_SCORE.md)
   [ "$count" -ge 4 ] && echo "OK: $count sections found" || echo "FAIL: only $count sections (expected >= 4)"
   ```
2. Run the gardening script and confirm it produces structured output:
   ```bash
   output=$(bash scripts/doc-gardening.sh 2>&1)
   rc=$?
   echo "$output"
   echo "Exit code: $rc"
   # Should produce structured output with summary counts
   echo "$output" | grep -qiE "(OK|MISSING|stale|mismatch|summary)" && echo "OK: structured output" || echo "FAIL: no structured output detected"
   # Exit code 0 = all clean, 1 = issues found (both are valid for verification)
   ```
3. Confirm the script checks pointer resolution:
   ```bash
   grep -q "claude.md" scripts/doc-gardening.sh && echo "OK" || echo "FAIL"
   ```
4. Confirm skill file exists:
   ```bash
   test -f .claude/skills/cleanup-sweep.md && echo "OK" || echo "FAIL"
   ```
5. Confirm `claude.md` reference:
   ```bash
   grep -q "QUALITY_SCORE" claude.md && echo "OK" || echo "FAIL"
   ```
6. Test graceful handling of missing upstream remote:
   ```bash
   git remote rename upstream upstream-bak 2>/dev/null || true
   output=$(bash scripts/doc-gardening.sh 2>&1)
   rc=$?
   git remote rename upstream-bak upstream 2>/dev/null || true
   echo "$output" | grep -qi "warn\|skip" && echo "OK: graceful warning" || echo "FAIL: no warning for missing upstream"
   # Script should NOT have crashed (rc should be 0 or 1, not 127/signal)
   [ "$rc" -le 1 ] && echo "OK: exit code $rc" || echo "FAIL: unexpected exit code $rc (crash?)"
   ```
7. Run `pnpm typecheck` and confirm zero errors
