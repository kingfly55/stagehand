# Milestone 3 — Golden Principles & Mechanical Enforcement

## Scope
Create mechanically enforced guardrails for the fork's hard constraints. Implement pre-commit hooks that prevent modifications to protected paths, block direct commits to `main`, and require `pnpm typecheck` to pass. Document all golden principles in a dedicated file with machine-enforceable counterparts.

## Files to Create/Modify
- `docs/dev/GOLDEN_PRINCIPLES.md` — new; fork-specific invariants with enforcement details
- `.husky/pre-commit` — new or modified; pre-commit hook script
- `.husky/pre-push` — new; pre-push hook for branch protection
- `package.json` — add husky/lint-staged config if not present
- `scripts/check-protected-paths.sh` — new; script to check staged files against protected paths
- `claude.md` — add pointer to `docs/dev/GOLDEN_PRINCIPLES.md`

## Implementation Steps
1. **Create `docs/dev/GOLDEN_PRINCIPLES.md`:**
   - List every hard constraint (these must match exactly with `claude.md` and the pre-commit enforcement):
     1. Never modify files in `packages/core/lib/v3/understudy/` (recursive — includes `a11y/`, subdirs) except `page.ts` **and `native/**`** (fork-only implementation directory)
     2. Never modify `packages/core/lib/v3/handlers/` (recursive — includes `handlerUtils/`) except to change `Page` type references to `IStagehandPage`
     3. All new implementation code goes in `packages/core/lib/v3/understudy/native/` (this directory is explicitly excluded from the protected path rule above)
     4. Never commit directly to `main` — use `native-base` or feature branches
     5. `pnpm typecheck` must exit 0 before any task is declared done
     6. `--no-verify` escape hatch: may be used for WIP commits or legitimate protected-path changes (e.g., merge commits), but the reason MUST be documented in the commit message
   - For each constraint, document: the principle, why it exists, how it's enforced (hook/CI), and remediation instructions if violated.

2. **Install husky** (if not already present):
   ```bash
   # Check if husky is already a devDependency
   if ! grep -q '"husky"' package.json; then
     pnpm add -Dw husky   # -w flag for workspace root
   fi
   # husky init creates .husky/ dir and a default pre-commit hook
   pnpm exec husky init
   ```
   **Note:** Use `pnpm add -Dw` (workspace root flag) since this is a pnpm workspace monorepo. Without `-w`, pnpm will refuse to install at root.
   **Note:** `husky init` may overwrite an existing `.husky/pre-commit`. Run init first, then create/overwrite hooks in subsequent steps.
   **Critical:** The root `package.json` already has `"prepare": "node packages/core/scripts/prepare.js"`. `husky init` will overwrite this with `"prepare": "husky"`. After running `husky init`, **restore the chained prepare script**:
   ```bash
   # Fix the prepare script to chain both:
   npm pkg set scripts.prepare="node packages/core/scripts/prepare.js && husky"
   ```
   Verify: `grep '"prepare"' package.json` should show both the original prepare.js call and husky.

3. **Create `scripts/check-protected-paths.sh`:**
   - Must start with `#!/usr/bin/env bash` and `set -euo pipefail`.
   - Reads `git diff --cached --name-only`
   - Rejects commits that modify files matching (use **recursive** patterns — the actual tree has subdirectories like `a11y/snapshot/` and `native/`):
     - `packages/core/lib/v3/understudy/**` (except `packages/core/lib/v3/understudy/page.ts` AND `packages/core/lib/v3/understudy/native/**`)
     - `packages/core/lib/v3/handlers/**` (except changes that ONLY modify `Page` → `IStagehandPage` type references; for simplicity, block all handler modifications and document `--no-verify` as the escape hatch for legitimate type-ref-only changes)
   - **CRITICAL: The `native/` directory is fork-only implementation code and MUST be allowed.** Without this exception, the hook blocks ALL fork development work. The actual files at `understudy/native/` (e.g., `PlaywrightNativePage.ts`, `PlaywrightNativeContext.ts`, `snapshot/`, `actions/`, `locator/`) are the primary deliverables of this fork.
   - **Pattern matching:** Use `case` statements or `grep -E` with anchored patterns, NOT shell globs which won't match recursively in a flat file list.
   - **Critical subshell bug:** Do NOT pipe into `while read` — the pipe creates a subshell, so `blocked=1` set inside the loop won't propagate to the parent shell, and the script will always exit 0. Use process substitution instead:
     ```bash
     # CORRECT: process substitution keeps variables in the current shell
     blocked=0
     while read -r file; do
       case "$file" in
         packages/core/lib/v3/understudy/page.ts) ;; # allowed exception
         packages/core/lib/v3/understudy/native/*) ;; # allowed — fork implementation dir
         packages/core/lib/v3/understudy/*) echo "BLOCKED: $file is protected. See docs/dev/GOLDEN_PRINCIPLES.md"; blocked=1 ;;
         packages/core/lib/v3/handlers/*) echo "BLOCKED: $file is protected. See docs/dev/GOLDEN_PRINCIPLES.md"; blocked=1 ;;
       esac
     done < <(git diff --cached --name-only)
     exit "$blocked"
     ```
     Note: `case` with `*` suffix matches recursively through path separators. The `native/*` rule MUST appear BEFORE the `understudy/*` rule because `case` uses first-match semantics.
   - **Alternative approach** (no process substitution needed): Use `grep` to detect blocked files and check the exit code:
     ```bash
     blocked_files=$(git diff --cached --name-only \
       | grep -E '^packages/core/lib/v3/(understudy/|handlers/)' \
       | grep -v '^packages/core/lib/v3/understudy/page.ts$' \
       | grep -v '^packages/core/lib/v3/understudy/native/' \
       || true)
     if [ -n "$blocked_files" ]; then
       echo "BLOCKED files:"; echo "$blocked_files"
       exit 1
     fi
     ```
   - **IMPORTANT:** Both approaches (case and grep) must exclude `understudy/native/**` — this is the fork's primary implementation directory.
   - Prints a clear error message with remediation: "BLOCKED: You modified [file]. This path is protected. See docs/dev/GOLDEN_PRINCIPLES.md for details."
   - **Exit code:** Must exit 0 if no protected files are staged (including when NO files are staged), exit 1 only if a protected path is found.
   - Make executable.
   - **Agent note:** Claude Code agents can use `git commit --no-verify` for legitimate exceptions, but MUST document the reason in the commit message.

4. **Create `.husky/pre-commit`:**
   ```bash
   # Protected path check (fast)
   bash scripts/check-protected-paths.sh || exit 1

   # Typecheck (~10s) — can be skipped with `git commit --no-verify` for WIP commits
   pnpm typecheck || exit 1
   ```
   **Husky version note:** Husky v9+ (installed via `husky init`) uses plain scripts without a shebang. If using husky v8 or earlier, add `#!/usr/bin/env sh` at the top. Check the installed version: `pnpm exec husky --version`.
   **Performance note:** `pnpm typecheck` adds ~10s per commit. This is intentional to catch type errors early. Agents making frequent WIP commits can use `--no-verify` but MUST run typecheck before declaring a task complete.

5. **Create `.husky/pre-push`:**
   ```bash
   branch=$(git rev-parse --abbrev-ref HEAD)
   if [ "$branch" = "main" ]; then
     echo "BLOCKED: Direct pushes to main are not allowed. See docs/dev/GOLDEN_PRINCIPLES.md"
     exit 1
   fi
   ```

6. **Update `claude.md`** to reference `docs/dev/GOLDEN_PRINCIPLES.md` in the documentation map.

7. **Test the hooks** by attempting a forbidden operation in a temporary branch.
