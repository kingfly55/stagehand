# Milestone 3 — Golden Principles & Mechanical Enforcement

## Scope
Create mechanically enforced guardrails for the fork's hard constraints. Implement pre-commit hooks that prevent modifications to protected paths, block direct commits to `main`, and require `pnpm typecheck` to pass. Document all golden principles in a dedicated file with machine-enforceable counterparts.

## Files to Create/Modify
- `docs/dev/GOLDEN_PRINCIPLES.md` — new; fork-specific invariants with enforcement details
- `.husky/pre-commit` — new or modified; pre-commit hook script
- `.husky/pre-push` — new; pre-push hook for branch protection
- `package.json` — add husky/lint-staged config if not present
- `scripts/check-protected-paths.sh` — new; script to check staged files against protected paths
- `claude.md` — add pointer to `docs/dev/GOLDEN_PRINCIPLES.md`

## Implementation Steps
1. **Create `docs/dev/GOLDEN_PRINCIPLES.md`:**
   - List every hard constraint:
     1. Never modify files in `packages/core/lib/v3/understudy/` except `page.ts`
     2. Never modify `packages/core/lib/v3/handlers/` except to change `Page` type references to `IStagehandPage`
     3. All new implementation code goes in `packages/core/lib/v3/understudy/native/`
     4. Never commit directly to `main` — use `native-base` or feature branches
     5. `pnpm typecheck` must exit 0 before any task is declared done
   - For each constraint, document: the principle, why it exists, how it's enforced (hook/CI), and remediation instructions if violated.

2. **Install husky** (if not already present):
   ```bash
   pnpm add -D husky
   pnpm exec husky init
   ```

3. **Create `scripts/check-protected-paths.sh`:**
   - Reads `git diff --cached --name-only`
   - Rejects commits that modify files matching:
     - `packages/core/lib/v3/understudy/*` (except `page.ts`)
     - Any path listed as protected in GOLDEN_PRINCIPLES.md
   - Prints a clear error message with remediation: "BLOCKED: You modified [file]. This path is protected. See docs/dev/GOLDEN_PRINCIPLES.md for details."
   - Make executable.

4. **Create `.husky/pre-commit`:**
   ```bash
   #!/bin/sh
   bash scripts/check-protected-paths.sh
   pnpm typecheck
   ```

5. **Create `.husky/pre-push`:**
   ```bash
   #!/bin/sh
   branch=$(git rev-parse --abbrev-ref HEAD)
   if [ "$branch" = "main" ]; then
     echo "BLOCKED: Direct pushes to main are not allowed. See docs/dev/GOLDEN_PRINCIPLES.md"
     exit 1
   fi
   ```

6. **Update `claude.md`** to reference `docs/dev/GOLDEN_PRINCIPLES.md` in the documentation map.

7. **Test the hooks** by attempting a forbidden operation in a temporary branch.

## Success Criteria
- [ ] `docs/dev/GOLDEN_PRINCIPLES.md` exists and lists all fork hard constraints
- [ ] `.husky/pre-commit` exists and is executable
- [ ] `.husky/pre-push` exists and is executable
- [ ] `scripts/check-protected-paths.sh` exists and is executable
- [ ] A staged change to a protected file (e.g., `packages/core/lib/v3/understudy/index.ts`) is rejected by the pre-commit hook with a clear error message
- [ ] Attempting to push to `main` is rejected by the pre-push hook
- [ ] `pnpm typecheck` passes with zero errors
- [ ] `claude.md` contains a reference to `docs/dev/GOLDEN_PRINCIPLES.md`

## Verification Steps
1. Confirm files exist and are executable:
   ```bash
   test -x .husky/pre-commit && echo "OK" || echo "FAIL"
   test -x .husky/pre-push && echo "OK" || echo "FAIL"
   test -x scripts/check-protected-paths.sh && echo "OK" || echo "FAIL"
   ```
2. Test protected path enforcement:
   ```bash
   # Create a temp file to simulate staging a protected path
   touch packages/core/lib/v3/understudy/test-forbidden.ts
   git add packages/core/lib/v3/understudy/test-forbidden.ts
   git commit -m "test: should be blocked" 2>&1 | grep -q "BLOCKED" && echo "OK: blocked" || echo "FAIL: not blocked"
   git reset HEAD packages/core/lib/v3/understudy/test-forbidden.ts
   rm packages/core/lib/v3/understudy/test-forbidden.ts
   ```
3. Test main branch protection:
   ```bash
   bash .husky/pre-push 2>&1 | grep -q "BLOCKED" || echo "OK: would block on main"
   ```
4. Confirm `docs/dev/GOLDEN_PRINCIPLES.md` contains the word "understudy" and "main"
5. Run `pnpm typecheck` and confirm zero errors
6. Confirm `claude.md` contains the string `GOLDEN_PRINCIPLES`
